\doxysubsection{Detailed Description}
Class representing a square board that contains colored cells. 

The class contains a grid which can display itself, draw lines on itself and count the number cells with a certain color on itself.

{\footnotesize\ttfamily void Board\+::\+Plot\+Line (\begin{DoxyParamCaption}\item[{const \mbox{\hyperlink{classParametricLine}{Parametric\+Line}} \&}]{line,  }\item[{const char \&}]{color }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [inline]}}



Draws a parametric line onto the board in a certain color. 

The function draws an imaginary line between the midpoints of the cells which form the endpoints of the parametric line and colors in all cells which this imaginary line touches with the input color.

This function implements the Amanatides and Woo Algorithm which is a raytracing algorithm described in detail in the following Git\+Hub document\+: \href{https://github.com/cgyurgyik/fast-voxel-traversal-algorithm/blob/master/overview/FastVoxelTraversalOverview.md}{\texttt{ https\+://github.\+com/cgyurgyik/fast-\/voxel-\/traversal-\/algorithm/blob/master/overview/\+Fast\+Voxel\+Traversal\+Overview.\+md}}

The main limitation of this algorithm is floating point arithmetic errors are cumulated over time. To address this and other issues created by floating point arithmetic errors a tolerance of around 1 $\ast$ 10$^\wedge$-\/12 has been introduced. However, the algorithm will start to produce errors if the line gets longer than around 1000 units.

\begin{DoxyPrecond}{Precondition}
The input line is valid / can be drawn on the board. 
\end{DoxyPrecond}
